hexo를 이용하여 Github에 블로그 만들기

Hexo를 이용하여 블로그를 만들기 위해 설치해야하는 필수 요소가 있습니다.

  • git
  • nvm
  • node.js
  • npm

hexo.io 사이트에서는 간단하게 5라인으로 요약이 되어 있지만, 실제로는 이렇게 5라인으로는 설치가 안되는 케이스가 더 많을 겁니다. 어쨋든…문제는 이 작업을 할 때 각 프로그램의 installer를 이용하여 설치를 하게 되면, 다른 컴퓨터로 이전하거나 OS 재설치할 때 조금 복잡한 면이 있다는 점입니다. 이런 점을 감안하여 여기서는 portable 프로그램을 이용하여 OS 재설치시에도 영향을 받지 않도록 D드라이브에 hexo를 설치해보도록 하겠습니다.


Git 설치 (portable 버전)

  1. https://git-scm.com/download/win 에서 PortableGit~.7z.exe 다운로드
  2. 실행후 D:\dev\Git 폴더지정하여 설치
  3. exe 파일은 삭제

nvm 설치 (noinstall 버전)

  1. https://github.com/coreybutler/nvm-windows/releases 에서 nvm-noinstall.zip 다운로드

  2. d:\dev 폴더에 압축풀고 폴더명을 nvm 으로 바꿈. zip 파일은 삭제

  3. nvm 폴더의 install.cmd 실행 (반드시 관리자 권한으로 실행)

    1. Enter the absolute path where the zip file is extracted/copied to: 라는 메세지가 나오면
      d:\dev\nvm 입력

    2. 성공: 지정한 값을 저장했습니다. 라는 메세지가 3번 나와야 함.

      • 관리자 권한으로 실행하지 않으면 실패 메세지 나옴
      • 지정한 폴더를 찾을 수 없다는 메세지가 나오면 d:\dev\nvm 폴더를 만든 이후 다시 install.cmd를 관리자 권한으로 실행

    3. setting.txt 수정할 수 있는 메모장이 나타나면 다음과 같이 입력 (d:\dev\nvm\setting.txt 로 저장됨)

    4. 환경 변수 구성 : 제어판> 고급 시스템 설정 보기> 시스템 속성> 고급> 환경 변수를 열고, 사용자 변수에 다음을 새로만들기 혹은 추가

      • 새로만들기
        • 변수이름 NVM_HOME, 변수값 D:\dev\nvm
        • 변수이름 NVM_SYMLINK, 변수값 D:\dev\nodejs
      • 추가 (다른 PATH가 이미 지정되어 있는 경우, 끝부분에 추가)
        • PATH : %NVM_HOME%; %NVM_SYMLINK%

node.js & npm 설치

  1. git-bash 실행 (프로그램 목록에서 git-bash 혹은 git-bash.exe 찾아서 실행)
  2. cd d:/dev/nvm
  3. nvm install latest (nvm install 이라고 입력하면 안됨) -> d:\dev\nvm\ 폴더에 node.js 와 npm이 설치됨
  4. nvm use 14.13.0 (설치된 nvm 사용 선언)
